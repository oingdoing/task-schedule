## 3. 화면 구성

### 3.1 페이지 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│  헤더: 타이틀                            [사용 방법]             │
├─────────────────────────────────────────────────────────────────┤
│  [← 이전 연도]    2026년     [다음 연도 →]                        │
├─────────────────────────────────────────────────────────────────┤
│  편집: [담당자 검색] [변경 취소] [날짜 수정] [명단 수정] [연도 삭제] │
├─────────────────────────────────────────────────────────────────┤
│  메인 영역: 연도별 테이블 뷰 (월|날짜|설거지|피청|커청|건청|간식|본교팀장|화장실|주 담당) │
└─────────────────────────────────────────────────────────────────┘
```

- **담당자 검색**: 편집 영역
- **변경 취소**: 교환 대상 선택 모드 해제 (확인 전)
- **날짜 수정·명단 수정·연도 삭제**: admin만 노출
- **사용 방법**: 헤더 우측, 항상 노출

### 3.2 담당자 교환

- 사람 이름 셀 클릭 → "담당자를 바꾸시겠습니까?" 확인
- 확인 시 → 교환 대상 선택 모드, 동일 업무 셀만 선택 가능
- 바꿀 셀 클릭 → "이 사람과 바꾸시겠습니까?" 확인
- 확인 시 → changeLog에 기록, 두 셀의 이름 교환

### 3.3 일정 확인 (날짜 팝업)

- **팝업 조건**: 일정 표시가 있거나 당번 없음인 날짜에 마우스 오버
- 우측 팝업으로 메모 표시, 3초 후 자동 숨김
- 스타일: 기령일(녹색), 공휴일/연휴(파란색), 기령일+주말/공휴일(강조)

### 3.4 주 담당 보기

- 주 담당 칸의 **보기** 버튼 클릭 → 해당 주의 담당자를 모아서 표시
- WeekDetailPanel 모달

## 4. 데이터 구조

### 4.1 전체 구조

```json
{
  "teams": {
    "설거지": { "1팀": ["이름1","이름2","이름3"], ... },
    "화장실청소": { "1팀": ["이름1","이름2"], ... }
  },
  "rosters": {
    "피청커청건청": ["이름1", "이름2", ...],
    "간식": ["이름1", "이름2", ...],
    "본교팀장주중": ["이름1", ...],
    "본교팀장주말": ["이름1", ...]
  },
  "schedule": [
    {
      "id": "slot-xxx",
      "date": "2026-03-05",
      "hasDuty": true,
      "scheduleMemoType": "기령일",
      "scheduleMemoDetail": "3월 정기",
      "dutyEnabled": { "피청": true, "커청": true, "건청": true, "간식": true, "본교팀장": true, "설거지": true }
    }
  ],
  "changeLog": [
    {
      "id": "log-xxx",
      "date": "2026-03-10",
      "dutyType": "피청",
      "cellA": { "slotId": "...", "date": "2026-03-04", "person": "김희권" },
      "cellB": { "slotId": "...", "date": "2026-03-11", "person": "김명세" }
    }
  ]
}
```

### 4.2 schedule (슬롯)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 고유 식별자 |
| date | string | YYYY-MM-DD |
| hasDuty | boolean | 당번 있음(true) / 당번 없음(false). 기본 true. |
| scheduleMemoType | string? | 일정 종류 (기령일, 경배일, 경신공부, 현장학습, 월례교학, 공휴일, 연휴, 기타) |
| scheduleMemoDetail | string? | 상세 사항 (공휴일/연휴/기타/기령일/경배일 선택 시) |
| dutyEnabled | object? | 당번 상세 체크박스 (피청, 커청, 건청, 간식, 본교팀장, 설거지) |
| noDutyMemo | string? | @deprecated scheduleMemoType/Detail 사용 권장 |

### 4.3 날짜 패턴

- 기본: **수·토·일** 3일
- 2026년: `CUSTOM_2026_START_DATE`(2026-02-25)부터, 2026-02-28 토요일 2슬롯
- 같은 날 2번: 허용

## 5. 일정 유형 및 당번 규칙

### 5.1 일정 유형 (ScheduleMemoType)

| 유형 | 상세 입력 | 비고 |
|------|-----------|------|
| 기령일 | ✓ | |
| 경배일 | ✓ | 설거지 당번 없음 |
| 경신공부 | ✗ | 설거지 당번 없음(경배일과 동일). UI 자동 처리는 경배일만 해당. |
| 현장학습 | ✗ | |
| 월례교학 | ✗ | |
| 공휴일 | ✓ | |
| 연휴 | ✓ | |
| 기타 | ✓ | |

### 5.2 당번 기본 규칙

- **기본**: 모든 일정은 당번 있음(hasDuty: true)
- **경배일만**: 설거지 당번 없음(dutyEnabled.설거지: false). 나머지 당번(피청, 커청 등) 유지.

### 5.3 경배일 vs 공휴일 중복

- **우선순위**: 경배일 우선 (1일, 15일 = 경배일)
- **겹치는 날** (1/1 신정, 3/1 삼일절, 8/15 광복절 등): 경배일 + 공휴일명 둘 다 기록
- 표시: `scheduleMemoType: '경배일'`, `scheduleMemoDetail: '신정'`

### 5.4 enrichSlotScheduleMemo

- `scheduleMemoType` 없을 때만 날짜 기준으로 자동 보완
- **사용자가 scheduleMemoType 지정한 경우**: dutyEnabled 덮어쓰지 않음 (당번 상세 체크 존중)
- 자동 보완: 1·15일→경배일, 공휴일→공휴일, 수요일→설거지 off

### 5.5 설거지 당번 없음 시 표시 (getScheduleLabelForNoDuty)

- **조건**: dutyEnabled.설거지 = false
- **표시 규칙** (우선순위 순):
  1. 경배일+상세 → `"경배일, {상세}"` (예: 경배일, 신정)
  2. scheduleMemoDetail 있음 → 상세 내용
  3. scheduleMemoType 있음 → 일정 이름

## 6. 로테이션 로직

### 6.1 업무별 방식

| 업무 | 단위 | 데이터 소스 | 비고 |
|------|------|-------------|------|
| 피청 | 1명/주 | rosters.피청커청건청 | 주 단위, 빈 행 건너뛰고 병합 |
| 커청 | 1명/주 | rosters.피청커청건청 | 동일 |
| 건청 | 2명/주 | rosters.피청커청건청 | 2명씩 겹치지 않게. 김보정부터(2026) |
| 간식 | 1명/주 | rosters.간식 | 김희권부터 |
| 본교팀장 | 주중/주말 | 본교팀장주중/주말 | 수요일=주중, 주말=토·일+연속일(금토일, 토일월 등) |
| 설거지 | 1팀/슬롯 | teams.설거지 | 토·일 슬롯만. dutyEnabled.설거지 false면 당번 없음 표시 |
| 화장실청소 | 1팀/월 | teams.화장실청소 | 월별, 홀수팀=제빙기/짝수팀=화장실 |

### 6.2 건청 (건관청소) 로테이션

- **형식**: 2명씩 겹치지 않게 (1주 a,b / 2주 c,d / 3주 e,f)
- **2026 커스텀**: 김보정부터 시작 → 첫 주 김보정·김주팔, 다음주 김보활·노영신

### 6.3 MERGE_ACROSS_EMPTY_DUTIES

- 피청, 커청, 건청, 간식: 빈 행(당번 없음) 건너뛰고 주 단위 병합

### 6.4 본교팀장 주말블록 (주말팀장)

- **기본**: 주말팀장은 토·일 담당
- **연속일 병합**: 앞뒤로 이어지는 날짜가 있으면 하나로 합침
  - 금토일(금·토·일) → 주말팀장 한 명이 전부 담당
  - 토일월(토·일·월) → 주말팀장 한 명이 전부 담당

### 6.5 화장실청소 표시

- 홀수 팀(1,3,5,7) → "제빙기" 뱃지
- 짝수 팀(2,4,6,8) → "화장실" 뱃지
- 셀 우측 상단 position: absolute

## 8. 기능 상세

### 8.1 날짜 수정 모달

- **범위**: 현재 연도만 수정
- **테이블**: 날짜 | 당번 유무 | 당번 상세 | 일정 | 편집
- **일정**: select(기령일, 경배일, 경신공부, 현장학습, 월례교학, 공휴일, 연휴, 기타) + input(상세)
- **상세 입력 활성화**: 5.1 표 참조 (기령일, 경배일, 공휴일, 연휴, 기타)
- **당번 없음 선택 시**: 당번 상세 체크박스 전부 해제
- **슬롯 추가**: 맨 위에 추가, 새 행 배경 #fff8e1 강조
- **저장 후**: sortSlots로 정렬, "저장되었습니다!" 알림

### 8.2 명단 수정 모달

- teams, rosters 편집
- rosters는 전역 공유 → 수정 시 전체 연도 일정에 공통 적용

### 8.3 다음 연도

- **다음 연도에 데이터 있음** → 해당 연도로 이동
- **다음 연도에 데이터 없음** → "다음 연도의 데이터를 생성하시겠습니까?" 확인
  - 확인 → 확인 코드 입력 모달
  - 코드 일치(YEAR_EXTEND_CONFIRM_CODE) → 다음 연도 슬롯 생성
- **생성**: 현재 연도 마지막 데이터 이어서, 해당 연도 공휴일 적용
- **공휴일**: KOREAN_HOLIDAYS_BY_YEAR (2026, 2027 등)

### 8.4 연도 삭제

- "연도 삭제" 버튼 → "{N}년도 데이터를 삭제하시겠습니까?" → 해당 연도 슬롯 전부 삭제

### 8.5 담당자 검색

- 검색어 포함 행만 필터
- 검색어와 일치하는 이름은 **굵게** 표시 (SearchHighlight)

### 8.6 사용 방법 모달

- 헤더 "사용 방법" 클릭 → 5개 섹션 안내
  1. 일정 확인: 날짜 마우스 오버
  2. 담당 변경: A 클릭 → 확인 → 원하는 사람 클릭
  3. 변경 취소: 굵은 이름 우측 점 → 변경 취소 버튼
  4. 주 담당: 보기 버튼
  5. 담당자 검색: 이름 검색

## 9. 접근 제어

### 9.1 입장 게이트

- **최초 입장**: 문서 코드 입력 필요
- **문서 코드**: `grant_document_access` RPC로 검증
- **역할 부여**: editor 또는 admin 반환
- **저장**: sessionStorage (documentId, role)

### 9.2 역할별 권한

| 역할 | 이름 변경·저장 | 날짜 수정 | 명단 수정 | 연도 삭제 | 다음 연도 생성 |
|------|----------------|-----------|-----------|-----------|----------------|
| editor | ✓ | ✗ | ✗ | ✗ | ✗ |
| admin | ✓ | ✓ | ✓ | ✓ | ✓ |

- **EmptyState**: admin만 "날짜 추가 시작" 버튼 노출
- **관리자로 입장하기**: admin 역할 부여용 코드 입력

### 9.3 편집 잠금

- **목적**: 동시 편집 충돌 방지
- **RPC**: `acquire_edit_lock`, `refresh_edit_lock`, `release_edit_lock`
- **획득 시점**: 날짜 수정 모달 열기, 명단 수정 모달 열기
- **해제 시점**: 저장 완료, 모달 닫기, 변경 취소, beforeunload, unmount

### 9.4 version 기반 충돌 방지 (추후 보완)

- **현재**: 편집 잠금으로 동시 편집 방지
- **보완 시**: `schedule_data.version`을 이용한 낙관적 잠금 추가 가능
- **적용 시점**: 사용자 수·동시 편집 가능성이 늘어날 때
- **구현 가이드**: [BACKEND_GUIDE.md](BACKEND_GUIDE.md) §6 참고

## 10. 변경 이력 및 취소

### 10.1 changeLog

- **저장**: Supabase schedule_data.payload.changeLog
- **표시**: `{날짜2} {사람B} ↔ {날짜1} {사람A}` 형식

### 10.2 변경 취소

- 바꾼 셀에 작은 점(●) 표시 → 클릭 시 툴팁 + "변경 취소" 버튼
- **취소 시**: 해당 로그 및 그 이후 로그 전부 제거 (`slice(0, index)`)
- 예: 1차·2차 교환이 있으면 2차 취소 시 1차만 남고 2차 변경만 원복
